本文档描述当前 OC Deck `0.2.0` 代码库。本文档不承诺所有未来构建版本、分发渠道、OpenCode Server、模型 Provider、Android 设备或网络运营商均以相同方式运行。

## OC Deck 服务和遥测

当前应用不运营由 OC Deck 托管的后端，也不有意包含产品分析或应用专用的崩溃报告服务。应用会连接用户选择或配置的服务。Android、应用商店、设备厂商、网络服务提供商或单独配置的服务，可能根据其各自条款收集信息。

## 连接用户配置的服务

OC Deck 会向用户选择的 OpenCode Server 配置发送请求。根据配置的模式，流量可能通过直连 HTTP(S)、SSH 本地端口转发或 frpc STCP visitor 传输。OpenCode Server 及其运营者、已配置的模型 Provider、隧道基础设施和网络中间方，可能处理或保留服务器地址、身份验证元数据、项目路径、prompt、session 内容、工具结果、附件和运行日志。

仅当 HTTPS 配置正确且验证有效时，它才能保护 HTTP 连接。SSH 或 STCP 会改变传输路径，但其本身并不决定 OpenCode Server 或模型 Provider 的数据保留或隐私惯例。

## Android 设备上存储的数据

OC Deck 在本地存储轻量级配置和偏好，包括服务器配置、最近项目、语言/主题选择、通知和声音设置，以及本地模型可见性偏好。运行时项目/session 状态主要保存在内存中。

敏感的服务器凭据、SSH 秘密信息、主机指纹和 STCP 秘密信息通过应用中由 Android Keystore 支持的凭据边界存储，普通配置则存储对凭据 alias 的引用。设备被攻破、不安全的备份、操作系统漏洞或实现缺陷仍可能影响机密性；本文档不作绝对安全保证。

移除服务器配置的预期效果是删除其本地配置和未被引用的凭据。按照 Android 的行为，卸载应用通常会删除应用私有的本地数据。这两项操作都不一定会删除 OpenCode Server、模型 Provider、SSH 服务器、frps 服务、备份系统或通知历史记录中已经存储的数据。

## 附件和项目文件

当用户选择手机本地附件时，OC Deck 会读取所选内容，并可能将其编码后作为 prompt 的一部分发送到已配置的 OpenCode Server。系统文件选择器会授予对所选项目的访问权限；在此流程中，OC Deck 无需广泛访问手机的全部存储空间。

浏览和选择服务器项目文件时，使用 OpenCode Server 的文件系统 API，而不是 Android 本地存储。用户将服务器项目文件加入 Composer 上下文后，OC Deck 只在内存草稿中保留经过校验的项目相对路径，并向已配置的 OpenCode Server 发送 `file://` 引用；服务端随后可能为 prompt 读取并展开该文件。OC Deck 不会把所选项目文件读成本机附件，也不会在 Android 端将其编码为 Base64。一个草稿最多包含 10 个服务器项目文件上下文。

## 通知和声音

如果已启用并获得 Android 许可，OC Deck 可以针对 Agent、权限或错误事件创建本地系统通知，并可以播放内置的通知声音。通知文本按设计使用经过本地化和脱敏的摘要，但根据 Android 和用户设置，session 标题或其他非秘密上下文仍可能显示在锁屏上。用户应适当配置锁屏隐私设置。

内置的 45 个声音文件是应用的本地资源。其来源和哈希记录在 `third_party/sources/opencode-audio.json` 中。

## 日志和诊断

Debug 构建版本可能会生成诊断日志。网络日志按设计应在非 Debug 构建中禁用，并经过脱敏处理，但用户在分享日志前仍应检查其内容。切勿分享原始凭据、Authorization 请求头、cookie、私钥、Provider 配置、非公开项目内容或完整的敏感响应。

## 用户选择

用户可以选择连接哪个服务器、是否配置 SSH 或 STCP、是否选择本地附件或服务器项目文件上下文，以及是否启用通知或声音。远程数据的访问、更正、保留和删除由相关 OpenCode Server 或 Provider 控制，必须与相应运营者处理。

## 变更

应用数据流的重大变更应反映在此文件及相关安全和架构文档中。请查阅随您所用构建版本一同提供的本文档版本。
