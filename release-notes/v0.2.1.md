## Release status / 发布状态

OC Deck `0.2.1` is a project-file navigation and copy-usability update over the public `0.2.0` release. It uses `VERSION_CODE=6` and requires no manual data migration.

OC Deck `0.2.1` 是基于公开 `0.2.0` 版本的项目文件导航与复制可用性更新，使用 `VERSION_CODE=6`，无需手工数据迁移。

OC Deck is an independent, community-maintained Android client. It is not developed, endorsed, supported, or affiliated with the OpenCode project or Anomaly. Original OC Deck code and documentation use the MIT License; bundled third-party components and assets retain their own licenses, notices, and trademark terms.

OC Deck 是独立社区维护的 Android 客户端，不是 OpenCode 项目或 Anomaly 官方开发、背书、支持或关联的产品。OC Deck 原创代码和文档使用 MIT License；内含第三方组件和资产继续受各自许可证、通知和商标条款约束。

## Highlights / 亮点

- Long-press a project file or directory row in Browse or Pick mode to copy its name, project-relative path, or normalized absolute server path. The menu does not open, expand, preview, or select the row. / 在浏览或选择模式中长按项目文件或目录行，可复制其名称、项目相对路径或规范化的服务端绝对路径；菜单不会打开、展开、预览或选择该行。
- Project-drawer destinations now navigate once before the drawer closes, improving responsiveness and preventing repeated taps during the close animation from launching multiple destinations. / 项目抽屉目标现在会在抽屉关闭前只导航一次，提升响应性，并避免关闭动画期间重复点击触发多个目标页面。
- Project switching from session details now produces a stable Back stack: the current project returns to its project home, and switching projects no longer leaves the previous session detail as the immediate Back destination. / 从会话详情切换项目现在会生成稳定的返回栈：选择当前项目会回到项目首页，切换项目后也不再把之前的会话详情留作紧接着的返回目标。
- While the project-file panel is open, system Back returns from preview to the file tree, closes the panel on the next press, and only then reaches the underlying project or session screen. / 项目文件面板打开时，系统返回键会先从预览回到文件树，再关闭面板，之后才到达底层项目页或会话页。

## Upgrade and migration / 升级与迁移

- No manual data migration is required. A same-signer Android replacement install from public `0.2.0` is expected to preserve app-private local data. / 无需手工数据迁移；使用相同签名从公开 `0.2.0` 执行 Android 覆盖安装，预计会保留应用私有本地数据。
- This release does not change DataStore formats, server profiles, credential references, Android permissions, OpenCode protocols, or the minimum Android version. / 本版本未修改 DataStore 格式、服务器配置、凭据引用、Android 权限、OpenCode 协议或最低 Android 版本。
- Record necessary non-secret connection details outside the app and keep another way to access the server because `0.x` storage, behavior, and compatibility may change. OC Deck currently has no supported settings or credential export/restore workflow. / 请在应用外记录必要的非敏感连接信息，并保留其他访问服务端的方式，因为 `0.x` 的存储、行为和兼容性仍可能变化。OC Deck 当前没有受支持的设置或凭据导出/恢复流程。
- Select the APK matching the device ABI. There is no universal APK or AAB in this release process. / 必须选择与设备 ABI 匹配的 APK；当前发布流程不提供 universal APK 或 AAB。
- OC Deck has no in-app updater. Compatible updates use Android's replacement-install flow; rollback requires uninstalling the current app and deletes app-private local data. / OC Deck 没有应用内更新器。兼容更新使用 Android 覆盖安装流程；回退需要卸载当前应用，并会删除应用私有本地数据。

## Compatibility / 兼容性

- Minimum Android version: Android 8.0 / API 26. / 最低 Android 版本：Android 8.0 / API 26。
- Published APK ABIs: `arm64-v8a`, `armeabi-v7a`, and `x86_64`. / 发布 APK ABI：`arm64-v8a`、`armeabi-v7a` 和 `x86_64`。
- The immutable GoMobile bridge coordinate is `io.github.ycfeng.ocdeck:frpc-stcp-visitor-gobridge:0.3.8-frp0.69.1-p1`. The coordinate changed because the AAR embeds the `0.2.1` legal inventory; Go, x/mobile, frp, Android API, and NDK versions are unchanged. / 不可变 GoMobile bridge 坐标为 `io.github.ycfeng.ocdeck:frpc-stcp-visitor-gobridge:0.3.8-frp0.69.1-p1`。该坐标因 AAR 内嵌 `0.2.1` 法律清单而变化；Go、x/mobile、frp、Android API 和 NDK 版本保持不变。
- OpenCode Server `1.17.7` remains `Observed only`, not a supported-version guarantee. Other server versions remain `Unknown`. / OpenCode Server `1.17.7` 仍仅标记为 `Observed only`，不构成受支持版本保证；其他服务端版本仍为 `Unknown`。
- Static release checks cover checksums, API/provenance, native-byte binding, expected ELF metadata, stripped state, 16KB alignment, and ABI isolation. Physical-device native loading, 16KB page-size operation, and a real STCP loop remain candidate-specific release gates. / 静态发布检查覆盖 checksum、API/provenance、native 字节绑定、预期 ELF metadata、stripped 状态、16KB 对齐和 ABI 隔离；真机 native load、16KB page-size 运行和真实 STCP 闭环仍是每个候选版本独立的发布门禁。

See the versioned compatibility matrix linked from the release preamble. / 详见发布固定前言链接的版本化兼容性矩阵。

## Known issues / 已知问题

- Provider management still requires broader real-server/provider and device validation, especially remote loopback OAuth, long-callback cancellation, and partial or unknown custom-provider outcomes. / Provider 管理仍需更广泛的真实 Server/Provider 和设备验证，重点包括远程 loopback OAuth、长 callback 取消，以及自定义 Provider 的 partial/unknown outcome。
- The standalone Review route is a placeholder; usable Git/session diff is in the session Changes tab. / 独立 Review route 仍为占位；可用 Git/session diff 位于会话 Changes tab。
- Model enabled/hidden settings are local filters and do not modify OpenCode Server configuration. / 模型 enabled/hidden 设置只是本机过滤，不会修改 OpenCode Server 配置。
- Shell mode, session sharing, workspace/worktree flows, and PTY/terminal are not implemented. / Shell mode、session share、workspace/worktree 和 PTY/terminal 尚未实现。
- Long-duration real-server disconnect/background behavior and STCP control-epoch recovery still require broader external validation. / 真实服务长时间断网、系统前后台行为和 STCP control epoch 恢复仍需更广泛外部验证。
- Android instrumentation tests are not currently run by CI. The new project-file Back-priority and clipboard-menu tests require a connected emulator or device, while rapid project-drawer navigation and complete Back-stack behavior still need broader physical-device validation. / Android instrumentation 测试当前仍未由 CI 执行。新增的项目文件返回优先级和剪贴板菜单测试需要连接 emulator 或设备；项目抽屉快速导航及完整返回栈行为仍需更广泛的真机验证。
- The project-file copy menu has no in-app success or failure message. Android system clipboard behavior and previews vary by OS version and device policy. / 项目文件复制菜单没有应用内成功或失败提示；Android 系统剪贴板行为和预览会因系统版本及设备策略而异。

## Security and privacy / 安全与隐私

- This release adds no authentication, network-transport, credential-storage, or server-operation changes. / 本版本未修改认证、网络传输、凭据存储或服务端操作。
- An explicit long press can place project file or directory names and paths on the Android system clipboard. File contents are not copied and no additional network request is made. Android or the device vendor may show clipboard previews or expose clipboard contents according to platform policy. / 用户可通过明确的长按操作，将项目文件或目录名称及路径写入 Android 系统剪贴板；不会复制文件内容，也不会额外发起网络请求。Android 或设备厂商可能根据平台策略显示剪贴板预览或允许访问剪贴板内容。
- Project paths can reveal server filesystem details. Review copied text before sharing it, and do not include private paths in public reports. / 项目路径可能暴露服务端文件系统细节；分享前请检查复制文本，不要在公开报告中包含私有路径。
- Connect only to OpenCode Server, SSH, and frps endpoints you trust. Server operators, model providers, and network intermediaries may receive project content, prompts, attachments, and metadata. / 只连接可信的 OpenCode Server、SSH 和 frps 端点。服务端运营者、模型 Provider 和网络中间方可能接收项目内容、prompt、附件和 metadata。
- Sensitive credentials are referenced through Android Keystore-backed storage and are redacted from supported logs and UI error paths. Provider keys are not shown in plaintext. / 敏感凭据通过 Android Keystore 支持的存储引用，并在受支持日志和 UI 错误路径中脱敏；Provider key 不明文回显。
- Do not include API keys, passwords, tokens, cookies, private keys, host fingerprints, provider headers, private server URLs, project content, prompts, or raw sensitive logs in public reports. Use `<redacted>`. / 公开报告不得包含 API key、密码、token、cookie、私钥、host fingerprint、provider header、私有服务器 URL、项目内容、prompt 或敏感原始日志，应使用 `<redacted>`。

## Downloads and verification / 下载与校验

The release workflow uploads exactly these assets: / Release workflow 精确上传以下 asset：

- `OCDeck_0.2.1_arm64-v8a.apk`
- `OCDeck_0.2.1_armeabi-v7a.apk`
- `OCDeck_0.2.1_x86_64.apk`
- `SHA256SUMS`

GitHub may additionally show automatic source archives. They are not APKs and cannot be installed. / GitHub 还可能显示自动源码归档；它们不是 APK，无法安装。

When only one ABI APK was downloaded, calculate its checksum and compare it with the same filename in `SHA256SUMS`: / 只下载一个 ABI APK 时，计算其 checksum，并与 `SHA256SUMS` 中同名文件比较：

```bash
sha256sum OCDeck_0.2.1_arm64-v8a.apk
grep 'OCDeck_0.2.1_arm64-v8a.apk' SHA256SUMS
```

On macOS use `shasum -a 256 OCDeck_0.2.1_arm64-v8a.apk`. On Windows PowerShell, use `Get-FileHash -Algorithm SHA256 .\OCDeck_0.2.1_arm64-v8a.apk` and locate the matching line with `Select-String -LiteralPath .\SHA256SUMS -SimpleMatch "OCDeck_0.2.1_arm64-v8a.apk"`. Use `sha256sum -c SHA256SUMS` only when all three listed APKs are present. Download only from <https://github.com/ycfeng/ocdeck-android>. / macOS 使用 `shasum -a 256 OCDeck_0.2.1_arm64-v8a.apk`。Windows PowerShell 使用 `Get-FileHash -Algorithm SHA256 .\OCDeck_0.2.1_arm64-v8a.apk`，再用 `Select-String -LiteralPath .\SHA256SUMS -SimpleMatch "OCDeck_0.2.1_arm64-v8a.apk"` 找到对应行。只有三个列出的 APK 都存在时才使用 `sha256sum -c SHA256SUMS`。只从 <https://github.com/ycfeng/ocdeck-android> 下载。
