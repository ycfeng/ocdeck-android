## Release status / 发布状态

OC Deck `0.2.3` is a long-session history loading and consistency update over the public `0.2.2` release. It uses `VERSION_CODE=8` and requires no manual data migration.

OC Deck `0.2.3` 是基于公开 `0.2.2` 版本的长会话历史加载与一致性更新，使用 `VERSION_CODE=8`，无需手工数据迁移。

## Highlights / 亮点

- Long session histories now load through cursor pages of 200 messages. Reaching the currently loaded top fetches one older page, while the first-user jump loads every remaining page before navigating to the true first user message. / 长会话历史现在通过每页 200 条消息的 cursor 分页加载；到达当前已加载顶部时获取一页更早历史，点击首条用户消息跳转时会先加载全部剩余页面，再定位到真正的第一条用户消息。
- Refresh and reconnect recovery now use Store-wide latest-generation checks, preserve concurrent SSE and optimistic updates, reconcile server-side deletions in the refreshed latest window, and maintain stable `(createdAt, id)` ordering across pages. / 刷新与重连恢复现在使用 Store 范围的最新 generation 校验，保留并发 SSE 与乐观更新，校准刷新后最新窗口中的服务端删除结果，并在分页间保持稳定的 `(createdAt, id)` 顺序。
- Pagination cursor state survives overlapping latest-page refreshes, safely restarts after disjoint refreshes to bridge gaps, rejects repeated cursor cycles, and is removed when its session is deleted. / 分页 cursor 状态可在最新页重叠刷新后保留，在不相交刷新后安全重启以补齐断层，拒绝重复 cursor 循环，并在对应会话删除时清理。

## Upgrade and migration / 升级与迁移

- No manual data migration is required. A same-signer Android replacement install from public `0.2.2` is expected to preserve app-private local data. / 无需手工数据迁移；使用相同签名从公开 `0.2.2` 执行 Android 覆盖安装，预计会保留应用私有本地数据。
- This release does not change DataStore formats, server profiles, credential references, Android permissions, OpenCode write protocols, or the minimum Android version. / 本版本未修改 DataStore 格式、服务器配置、凭据引用、Android 权限、OpenCode 写操作协议或最低 Android 版本。
- Record necessary non-secret connection details outside the app and keep another way to access the server because `0.x` storage, behavior, and compatibility may change. OC Deck currently has no supported settings or credential export/restore workflow. / 请在应用外记录必要的非敏感连接信息，并保留其他访问服务端的方式，因为 `0.x` 的存储、行为和兼容性仍可能变化。OC Deck 当前没有受支持的设置或凭据导出/恢复流程。
- Select the APK matching the device ABI. There is no universal APK or AAB in this release process. / 必须选择与设备 ABI 匹配的 APK；当前发布流程不提供 universal APK 或 AAB。
- OC Deck has no in-app updater. Compatible updates use Android's replacement-install flow; rollback requires uninstalling the current app and deletes app-private local data. / OC Deck 没有应用内更新器。兼容更新使用 Android 覆盖安装流程；回退需要卸载当前应用，并会删除应用私有本地数据。

## Compatibility / 兼容性

- Minimum Android version: Android 8.0 / API 26. / 最低 Android 版本：Android 8.0 / API 26。
- Published APK ABIs: `arm64-v8a`, `armeabi-v7a`, and `x86_64`. / 发布 APK ABI：`arm64-v8a`、`armeabi-v7a` 和 `x86_64`。
- The immutable GoMobile bridge coordinate is `io.github.ycfeng.ocdeck:frpc-stcp-visitor-gobridge:0.3.10-frp0.69.1-p1`. The coordinate changed because the AAR embeds the `0.2.3` legal inventory; Go, x/mobile, frp, Android API, and NDK versions are unchanged. / 不可变 GoMobile bridge 坐标为 `io.github.ycfeng.ocdeck:frpc-stcp-visitor-gobridge:0.3.10-frp0.69.1-p1`。该坐标因 AAR 内嵌 `0.2.3` 法律清单而变化；Go、x/mobile、frp、Android API 和 NDK 版本保持不变。
- OpenCode Server `1.17.7` remains `Observed only`, not a supported-version guarantee. Other server versions remain `Unknown`. / OpenCode Server `1.17.7` 仍仅标记为 `Observed only`，不构成受支持版本保证；其他服务端版本仍为 `Unknown`。
- Static release checks cover checksums, API/provenance, native-byte binding, expected ELF metadata, stripped state, 16KB alignment, and ABI isolation. Physical-device native loading, 16KB page-size operation, and a real STCP loop remain candidate-specific release gates. / 静态发布检查覆盖 checksum、API/provenance、native 字节绑定、预期 ELF metadata、stripped 状态、16KB 对齐和 ABI 隔离；真机 native load、16KB page-size 运行和真实 STCP 闭环仍是每个候选版本独立的发布门禁。

See the versioned compatibility matrix linked from the release preamble. / 详见发布固定前言链接的版本化兼容性矩阵。

## Known issues / 已知问题

- Loading the true first user message intentionally requests all remaining history pages and can take time on very long sessions. Per-page response limits do not cap the memory used by the complete accumulated history. / 定位真正的第一条用户消息会按设计请求全部剩余历史页，在极长会话中可能耗时；逐页响应上限不限制完整累计历史占用的内存。
- Provider management still requires broader real-server/provider and device validation, especially remote loopback OAuth, long-callback cancellation, and partial or unknown custom-provider outcomes. / Provider 管理仍需更广泛的真实 Server/Provider 和设备验证，重点包括远程 loopback OAuth、长 callback 取消，以及自定义 Provider 的 partial/unknown outcome。
- The standalone Review route is a placeholder; usable Git/session diff is in the session Changes tab. / 独立 Review route 仍为占位；可用 Git/session diff 位于会话 Changes tab。
- Model enabled/hidden settings are local filters and do not modify OpenCode Server configuration. / 模型 enabled/hidden 设置只是本机过滤，不会修改 OpenCode Server 配置。
- Shell mode, session sharing, workspace/worktree flows, and PTY/terminal are not implemented. / Shell mode、session share、workspace/worktree 和 PTY/terminal 尚未实现。
- Long-duration real-server disconnect/background behavior and STCP control-epoch recovery still require broader external validation. / 真实服务长时间断网、系统前后台行为和 STCP control epoch 恢复仍需更广泛外部验证。
- Android instrumentation tests are not currently run by CI. Long-history pagination, automatic top loading, and complete first-user navigation still require broader connected-device validation. / Android instrumentation 测试当前仍未由 CI 执行；长历史分页、到顶自动加载和完整首条用户消息定位仍需更广泛的连接设备验证。

## Security and privacy / 安全与隐私

- Session-message requests remain read-only and use the existing authenticated OpenCode connection. This release adds `before` cursor queries and reads `X-Next-Cursor`; it does not add credentials, request bodies, or server mutations. / 会话消息请求仍为只读，并使用现有已认证 OpenCode 连接；本版本新增 `before` cursor 查询并读取 `X-Next-Cursor`，未新增凭据、请求 body 或服务端写操作。
- Every history page retains separate 64 MiB encoded response-body and 64 MiB decoded-entity limits, body-free non-2xx handling, callback-thread streaming decode, and cancellation propagation. / 每个历史页继续使用独立的 64 MiB encoded response-body 与 64 MiB decoded-entity 上限、非 2xx 无 body 处理、callback 线程流式解码和取消传播。
- Connect only to OpenCode Server, SSH, and frps endpoints you trust. Server operators, model providers, and network intermediaries may receive project content, prompts, attachments, and metadata. / 只连接可信的 OpenCode Server、SSH 和 frps 端点。服务端运营者、模型 Provider 和网络中间方可能接收项目内容、prompt、附件和 metadata。
- Sensitive credentials are referenced through Android Keystore-backed storage and are redacted from supported logs and UI error paths. Provider keys are not shown in plaintext. / 敏感凭据通过 Android Keystore 支持的存储引用，并在受支持日志和 UI 错误路径中脱敏；Provider key 不明文回显。
- Do not include API keys, passwords, tokens, cookies, private keys, host fingerprints, provider headers, private server URLs, project content, prompts, or raw sensitive logs in public reports. Use `<redacted>`. / 公开报告不得包含 API key、密码、token、cookie、私钥、host fingerprint、provider header、私有服务器 URL、项目内容、prompt 或敏感原始日志，应使用 `<redacted>`。

GitHub may additionally show automatic source archives. They are not APKs and cannot be installed. / GitHub 还可能显示自动源码归档；它们不是 APK，无法安装。

When only one ABI APK was downloaded, calculate its checksum and compare it with the same filename in `SHA256SUMS`: / 只下载一个 ABI APK 时，计算其 checksum，并与 `SHA256SUMS` 中同名文件比较：

```bash
sha256sum OCDeck_0.2.3_arm64-v8a.apk
grep 'OCDeck_0.2.3_arm64-v8a.apk' SHA256SUMS
```

On macOS use `shasum -a 256 OCDeck_0.2.3_arm64-v8a.apk`. On Windows PowerShell, use `Get-FileHash -Algorithm SHA256 .\OCDeck_0.2.3_arm64-v8a.apk` and locate the matching line with `Select-String -LiteralPath .\SHA256SUMS -SimpleMatch "OCDeck_0.2.3_arm64-v8a.apk"`. Use `sha256sum -c SHA256SUMS` only when all three listed APKs are present. Download only from <https://github.com/ycfeng/ocdeck-android>. / macOS 使用 `shasum -a 256 OCDeck_0.2.3_arm64-v8a.apk`。Windows PowerShell 使用 `Get-FileHash -Algorithm SHA256 .\OCDeck_0.2.3_arm64-v8a.apk`，再用 `Select-String -LiteralPath .\SHA256SUMS -SimpleMatch "OCDeck_0.2.3_arm64-v8a.apk"` 找到对应行。只有三个列出的 APK 都存在时才使用 `sha256sum -c SHA256SUMS`。只从 <https://github.com/ycfeng/ocdeck-android> 下载。
