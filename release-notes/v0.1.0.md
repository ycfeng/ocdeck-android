## Highlights / 亮点

- Use the unified OC Deck console-control-panel logo across launcher, splash, notifications, and in-app identity surfaces. / 在启动器、启动画面、通知和应用内身份标识界面统一使用 OC Deck 控制台面板 Logo。
- Connect to a user-provided OpenCode Server through direct HTTP(S), SSH local forwarding, or an embedded frpc STCP visitor. / 支持通过 HTTP(S) 直连、SSH 本地转发或内置 frpc STCP visitor 连接用户自备的 OpenCode Server。
- Browse projects and sessions, lazily create sessions on first send, send asynchronous prompts, and manage rename/archive/delete/revert/unrevert flows. / 支持项目与会话浏览、首次发送时懒创建会话、异步 prompt，以及重命名/归档/删除/revert/unrevert。
- Synchronize global and project state through bounded SSE readers with reconnect and REST snapshot reconciliation. / 通过有界 SSE reader、自动重连和 REST 快照校准同步全局与项目状态。
- Use Slash commands, agent mentions, agent/model/variant pickers, local-device attachments, server project-file contexts, permission/question responses, context usage, and the session Changes tab. / 支持 Slash command、agent mention、agent/model/variant picker、手机本地附件、服务端项目文件上下文、permission/question 响应、上下文用量和会话 Changes tab。
- Browse and search server project files with read-only bounded text and image previews, then select up to 10 whole files as live Composer context. / 支持服务端项目文件只读浏览、搜索和有边界的文本/图片预览，并可选择最多 10 个完整文件作为 Composer 实时上下文。
- Use English or Simplified Chinese UI, theme preferences, Android notifications, sounds, and background-run guidance. / 支持英文或简体中文 UI、主题偏好、Android 通知、音效和后台运行引导。
- Include the fixed GoMobile bridge `io.github.ycfeng.ocdeck:frpc-stcp-visitor-gobridge:0.3.2-frp0.69.1-p1`, based on frp `v0.69.1-p1`, with pinned inputs and auditable provenance. / 包含固定 GoMobile bridge `io.github.ycfeng.ocdeck:frpc-stcp-visitor-gobridge:0.3.2-frp0.69.1-p1`，基于 frp `v0.69.1-p1`，并固定输入与提供可审计 provenance。

## Upgrade and migration / 升级与迁移

- This is the prepared first public release, so there is no supported prior public OC Deck version or replacement-update path to validate. Record necessary non-secret connection details outside the app and keep another way to access the server because `0.x` storage, behavior, and compatibility may change. OC Deck currently has no supported settings or credential export/restore workflow. / 这是预备的首个公开版本，因此不存在受支持的旧公开 OC Deck 版本或可验证的覆盖升级路径。请在应用外记录必要的非敏感连接信息，并保留其他访问服务端的方式，因为 `0.x` 的存储、行为和兼容性仍可能变化。OC Deck 当前没有受支持的设置或凭据导出/恢复流程。
- A one-time local migration removes only a historical auto-generated direct profile that still exactly matches the old `local` / `Localhost` / loopback `:4096` defaults. User-modified names, URLs, authentication, SSH, or STCP settings are retained. / 一次性本机迁移只删除仍完全匹配旧 `local` / `Localhost` / loopback `:4096` 默认值的历史自动生成直连配置。用户修改过名称、URL、认证、SSH 或 STCP 的配置会保留。
- Select the APK matching the device ABI. There is no universal APK or AAB in this release process. / 必须选择与设备 ABI 匹配的 APK；当前发布流程不提供 universal APK 或 AAB。
- OC Deck has no in-app updater. A future compatible update should use Android's replacement install flow. Normal Release rollback requires uninstalling the current app, deletes app-private local data, and then installs the older APK. Follow the versioned installation guide linked from the release preamble. / OC Deck 没有应用内更新器。未来兼容更新应使用 Android 覆盖安装流程。普通 Release 回退需要先卸载当前应用，这会删除应用私有本地数据，然后再安装旧 APK。请遵循发布前言中固定到本版本的安装指南。

## Compatibility / 兼容性

- Minimum Android version: Android 8.0 / API 26. / 最低 Android 版本：Android 8.0 / API 26。
- Published APK ABIs: `arm64-v8a`, `armeabi-v7a`, and `x86_64`. / 发布 APK ABI：`arm64-v8a`、`armeabi-v7a` 和 `x86_64`。
- OpenCode Server `1.17.7` is `Observed only`: it was seen in one documentation environment and is not a supported-version guarantee or a complete regression result. Other server versions remain `Unknown`. / OpenCode Server `1.17.7` 只能标记为 `Observed only`：它来自一次文档观察环境，不是受支持版本保证，也不是完整回归结果。其他服务端版本仍为 `Unknown`。
- Static bridge and APK checks cover checksums, API/provenance, expected native metadata, stripped state, 16KB ELF alignment, ABI isolation, and native-byte binding. Physical-device native load, a 16KB page-size device, and a real STCP end-to-end loop still require external evidence and were not completed when this note was prepared. / 静态 bridge 与 APK 检查覆盖 checksum、API/provenance、预期 native metadata、stripped 状态、16KB ELF 对齐、ABI 隔离和 native 字节绑定。本说明准备时，真机 native load、16KB page-size 真机和真实 STCP 端到端闭环仍缺少外部验证。

See the versioned compatibility matrix linked from the release preamble. / 详见发布固定前言链接的版本化兼容性矩阵。

## Known issues / 已知问题

- Provider authentication, disconnect, OAuth, and custom-provider persistence remain prototypes. / Provider 认证、断开、OAuth 和自定义 Provider 持久化仍是原型。
- The standalone Review route is a placeholder; usable Git/session diff is in the session Changes tab. / 独立 Review route 仍为占位；可用 Git/session diff 位于会话 Changes tab。
- Model enabled/hidden settings are local filters and do not modify OpenCode Server configuration. / 模型 enabled/hidden 设置只是本机过滤，不会修改 OpenCode Server 配置。
- Shell mode, session sharing, workspace/worktree flows, and PTY/terminal are not implemented. / Shell mode、session share、workspace/worktree 和 PTY/terminal 尚未实现。
- Long-duration real-server disconnect/background behavior and STCP control-epoch recovery still require broader external validation. / 真实服务长时间断网、系统前后台行为和 STCP control epoch 恢复仍需更广泛外部验证。
- Required physical-device native load, 16KB page-size, and real STCP release gates remain incomplete until maintainers record passing evidence. / 要求的真机 native load、16KB page-size 和真实 STCP 发布门禁，在维护者记录通过证据前仍视为未完成。

## Security and privacy / 安全与隐私

- Connect only to OpenCode Server, SSH, and frps endpoints you trust. Server operators, model providers, and network intermediaries may receive project content, prompts, attachments, and metadata. / 只连接可信的 OpenCode Server、SSH 和 frps 端点。服务端运营者、模型 Provider 和网络中间方可能接收项目内容、prompt、附件和 metadata。
- Sensitive credentials are referenced through Android Keystore-backed storage and are redacted from supported logs and UI error paths. Provider keys are not shown in plaintext. / 敏感凭据通过 Android Keystore 支持的存储引用，并在受支持日志和 UI 错误路径中脱敏；Provider key 不明文回显。
- Server URLs reject userinfo, query, and fragment components; SSH host keys are verified before user authentication. / 服务器 URL 拒绝 userinfo、query 和 fragment；SSH host key 在用户认证前校验。
- Session-message and SSE transports have endpoint-specific hard input limits. This does not mean every REST endpoint has completed bounded-input auditing. / Session messages 和 SSE transport 具有端点专项硬上限；这不代表所有 REST endpoint 都已完成有界输入审计。
- Do not include API keys, passwords, tokens, cookies, private keys, host fingerprints, provider headers, private server URLs, project content, prompts, or raw sensitive logs in public reports. Use `<redacted>`. / 公开报告不得包含 API key、密码、token、cookie、私钥、host fingerprint、provider header、私有服务器 URL、项目内容、prompt 或敏感原始日志，应使用 `<redacted>`。

## Downloads and verification / 下载与校验

The release workflow is expected to upload exactly these assets: / Release workflow 预期精确上传以下 asset：

- `OCDeck_0.1.0_arm64-v8a.apk`
- `OCDeck_0.1.0_armeabi-v7a.apk`
- `OCDeck_0.1.0_x86_64.apk`
- `SHA256SUMS`

GitHub may additionally show automatic `Source code (zip)` and `Source code (tar.gz)` archives. They are not APKs and cannot be installed. / GitHub 还可能显示自动生成的 `Source code (zip)` 和 `Source code (tar.gz)`；它们不是 APK，无法安装。

When only one ABI APK was downloaded, calculate that file's checksum and compare it with the same filename in `SHA256SUMS`: / 只下载一个 ABI APK 时，计算该文件的 checksum，并与 `SHA256SUMS` 中同名文件比较：

```bash
sha256sum OCDeck_0.1.0_arm64-v8a.apk
grep 'OCDeck_0.1.0_arm64-v8a.apk' SHA256SUMS
```

On macOS use `shasum -a 256 OCDeck_0.1.0_arm64-v8a.apk`. On Windows PowerShell, use `Get-FileHash -Algorithm SHA256 .\OCDeck_0.1.0_arm64-v8a.apk` and locate the matching line with `Select-String -LiteralPath .\SHA256SUMS -SimpleMatch "OCDeck_0.1.0_arm64-v8a.apk"`. `sha256sum -c SHA256SUMS` is suitable when all three listed APKs are present. Download only from the canonical repository <https://github.com/ycfeng/ocdeck-android>; do not install APKs from an untrusted mirror. / macOS 使用 `shasum -a 256 OCDeck_0.1.0_arm64-v8a.apk`。Windows PowerShell 使用 `Get-FileHash -Algorithm SHA256 .\OCDeck_0.1.0_arm64-v8a.apk`，再用 `Select-String -LiteralPath .\SHA256SUMS -SimpleMatch "OCDeck_0.1.0_arm64-v8a.apk"` 找到对应行。只有列出的三个 APK 都存在时，才适合使用 `sha256sum -c SHA256SUMS`。只从 canonical 仓库 <https://github.com/ycfeng/ocdeck-android> 下载，不要安装不可信镜像提供的 APK。
